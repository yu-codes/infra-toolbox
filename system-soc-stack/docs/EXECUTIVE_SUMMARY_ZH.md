# 安全監控系統業務總覽報告

## 一、系統簡介

本系統採用 Wazuh SIEM 4.14.5 + ClamAV 反病毒組合，用於持續監控伺服器環境，及時發現和應對安全威脅。

---

## 二、核心服務介紹

### 2.1 Wazuh - 安全信息和事件管理系統

**Wazuh 的功能：**
- 企業級的安全信息和事件管理系統（SIEM）
- 監控伺服器上發生的安全相關事件
- 根據預設規則分析事件並識別異常行為
- 當檢測到告警條件時發出警報

**Wazuh 的核心能力：**
- **事件收集**：收集伺服器、應用程序和系統的安全相關事件
- **規則分析**：根據預設的安全規則分析事件，識別異常行為
- **告警通知**：生成告警並通知相關人員
- **數據展示**：通過儀表板展示安全信息
- **集中管理**：統一管理多臺伺服器的監控

### 2.2 ClamAV - 反病毒和惡意軟體掃描系統

**ClamAV 的功能：**
- 反病毒和惡意軟體防護引擎
- 掃描伺服器上的文件，查找已知的病毒和惡意程序
- 自動更新病毒定義庫
- 輕量級設計，對伺服器性能影響較小

**ClamAV 的核心能力：**
- **持續防護**：監控伺服器文件系統
- **定期更新**：每 2 小時自動更新病毒定義庫
- **定期掃描**：每天自動執行全碟掃描
- **檢測報警**：檢測到病毒或惡意軟體時發出報警
- **詳細記錄**：完整記錄所有掃描結果

### 2.3 儀表板

提供 Web 界面用於：
- 查看伺服器的安全狀態
- 查詢和分析安全事件
- 追蹤被監控的伺服器和代理程序
- 生成安全報告

## 三、監控和掃描功能清單

### 3.1 系統事件監控

| 功能 | 監控範圍 | 目的 |
|------|----------|------|
| **用戶訪問監控** | 用戶登入、登出、失敗嘗試 | 記錄用戶訪問活動 |
| **特權操作監控** | 管理員 (sudo) 操作記錄 | 追蹤管理員操作 |
| **系統變更監控** | 軟件安裝、配置修改 | 記錄系統變更 |
| **進程監控** | 運行的應用程序和服務 | 監控進程活動 |
| **網絡活動監控** | 網絡連接嘗試 | 監控網絡連接 |

### 3.2 病毒和惡意軟體掃描

| 功能 | 掃描範圍 | 執行頻率 | 說明 |
|------|----------|----------|------|
| **全碟掃描** | 整個伺服器文件系統 | 每天自動（凌晨 2 點） | 掃描整個磁碟 |
| **即時掃描** | 新建/修改的文件 | 實時監控 | 檢測新增或修改的文件 |
| **病毒庫更新** | 最新威脅定義 | 每 2 小時自動更新 | 更新最新的病毒定義 |

### 3.3 高級安全功能

| 功能 | 說明 | 用途 |
|------|------|------|
| **檔案完整性監控 (FIM)** | 監控關鍵系統文件是否被修改 | 檢測系統文件篡改 |
| **已知漏洞掃描** | 檢測伺服器是否存在已公開的安全漏洞 | 識別已知漏洞 |
| **異常行為檢測** | 識別不符合正常模式的活動 | 發現可疑行為 |
| **審計追蹤** | 記錄所有重要操作和變更 | 支持安全審計 |

## 四、系統可監控的安全威脅

### 4.1 主要安全威脅類型

本系統可以監控和檢測以下安全威脅：

| 威脅類型 | 具體表現 | 監控方式 |
|----------|----------|----------|
| **惡意軟體感染** | 病毒、木馬、勒索軟體進入系統 | ClamAV 掃描和檢測 |
| **未授權訪問** | 黑客或內部人員非法登入 | 用戶訪問監控和異常檢測 |
| **數據洩露** | 敏感文件被非法復制或訪問 | 文件訪問監控和審計日誌 |
| **系統篡改** | 關鍵文件被修改或刪除 | 文件完整性監控 (FIM) |
| **特權濫用** | 管理員賬號被非法使用 | 特權操作監控和審計 |
| **已知漏洞利用** | 攻擊者利用已發布的系統漏洞 | 漏洞掃描和識別 |
| **隱藏威脅** | 後門程序、跳板機潛伏在系統 | 持續監控和異常行為檢測 |

### 4.2 監控覆蓋範圍

- **系統級威脅**：監控作業系統層級的異常
- **應用級威脅**：監控應用程序層級的異常
- **文件級威脅**：監控文件訪問和修改
- **網絡級威脅**：監控異常網絡連接
- **用戶級威脅**：監控訪問權限異常

## 五、系統輸出日誌清單

### 5.1 主要日誌類型

#### 1. **安全告警日誌**
- **包含內容**：系統發現的所有安全事件和告警
- **重要性**：最高 - 需要立即關注
- **示例**：
  - 發現病毒或惡意軟體
  - 檢測到異常登入嘗試
  - 系統文件被修改
  - 高風險操作執行

#### 2. **身份認證日誌**
- **包含內容**：所有用戶登入、登出和身份驗證嘗試
- **用途**：審計訪問控制
- **記錄信息**：
  - 用戶名和登入時間
  - 登入結果（成功/失敗）
  - 登入來源（IP 地址）
  - 失敗原因（如密碼錯誤）

#### 3. **特權操作日誌**
- **包含內容**：管理員 (sudo) 執行的所有操作
- **用途**：追蹤管理活動，防止濫用
- **記錄信息**：
  - 執行的命令
  - 執行時間和用戶
  - 執行結果

#### 4. **防毒掃描日誌**
- **包含內容**：ClamAV 掃描結果詳情
- **用途**：檢測和追蹤惡意軟體
- **記錄信息**：
  - 掃描時間和範圍
  - 檢測到的威脅數量
  - 威脅位置和類型
  - 處理操作（隔離/刪除）

#### 5. **文件完整性監控日誌**
- **包含內容**：關鍵系統文件的修改記錄
- **用途**：檢測系統文件篡改
- **記錄信息**：
  - 被修改的文件路徑
  - 修改時間
  - 修改類型（新增/刪除/修改）
  - 修改者信息

#### 6. **系統事件日誌**
- **包含內容**：伺服器運行中的常規事件
- **用途**：了解系統運行狀態
- **記錄信息**：
  - 軟件安裝和卸載
  - 配置改變
  - 服務啟停
  - 系統重啟

#### 7. **漏洞掃描日誌**
- **包含內容**：已知漏洞檢測結果
- **用途**：識別需要修復的漏洞
- **記錄信息**：
  - 漏洞编号 (CVE)
  - 風險等級
  - 受影響的軟件
  - 修復建議

### 5.2 日誌保留策略

| 日誌類型 | 保留期限 | 訪問方式 |
|----------|----------|----------|
| 告警日誌 | 90 天 | 儀表板 + 備份存儲 |
| 審計日誌 | 365 天 | 儀表板 + 檔案存儲 |
| 掃描日誌 | 30 天 | 儀表板 + 本地文件 |

## 六、額外的重要信息

### 6.1 系統健康監控

除了安全監控，系統還會跟蹤以下運行狀況：

- **系統資源使用**：CPU、內存、磁碟空間
- **服務可用性**：所有監控服務是否正常運行
- **數據庫連接**：日誌存儲系統是否正常工作
- **病毒庫版本**：防毒定義是否是最新的

### 6.2 告警等級說明

系統根據事件的嚴重程度分為不同的告警等級：

| 等級 | 說明 | 需要行動 |
|------|------|----------|
| **低** | 信息性事件，無直接威脅 | 備註即可 |
| **中** | 需要關注的異常行為 | 監控並調查 |
| **高** | 可能的安全事件 | 立即調查和響應 |
| **關鍵** | 高級別告警 | 立即採取應對措施 |

### 6.3 建議的最佳實踐

#### 日常管理
- 每周查看一次安全儀表板摘要
- 監控關鍵告警通知
- 定期檢查掃描結果

#### 定期維護
- 每月生成一份安全報告
- 定期更新系統和補丁
- 審查訪問日誌記錄

#### 應急響應
- 建立安全事件響應流程
- 為關鍵告警設置自動通知
- 定期進行安全演練

### 6.4 系統效能影響

本系統設計以降低對伺服器性能的影響：

- 防毒掃描在非高峰時段執行（凌晨 2 點）
- 日誌分析在後臺進行
- 整體系統開銷預計低於 5% 的 CPU 和內存

### 6.5 與現有系統的整合

本系統可以：
- 與現有的備份系統協同運行
- 與其他安全工具集成
- 支持多伺服器中央監控
- 導出日誌用於第三方分析

### 6.6 合規性和審計

該系統支持以下合規性要求：

- **日誌記錄**：完整的審計追蹤
- **訪問控制**：用戶和特權操作監控
- **定期掃描**：自動病毒和漏洞掃描
- **報告生成**：符合監管要求的報告格式
